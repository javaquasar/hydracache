# HydraCache 0.1.0 Release Notes

HydraCache 0.1.0 is the first public release of the project. It provides a
small local async cache runtime for Rust applications, plus a separate core
crate with shared types, options, errors, stats, and codec abstractions.

This release is intentionally local-only. It does not include SQL adapters,
distributed invalidation, or single-flight loader deduplication yet. The goal of
the first release is to establish a practical embedded cache API that can grow
into those features without changing the basic user experience too much.

## Published Crates

The release is split into two crates:

- `hydracache-core`: shared types used by the runtime.
- `hydracache`: the user-facing local async cache runtime.

`hydracache` depends on `hydracache-core`, so `hydracache-core` must be
published first.

## Minimum Supported Rust Version

HydraCache 0.1.0 uses:

```toml
rust-version = "1.75"
edition = "2021"
license = "Apache-2.0"
```

## Installation

For application code, depend on the user-facing crate:

```toml
[dependencies]
hydracache = "0.1"
serde = { version = "1", features = ["derive"] }
tokio = { version = "1", features = ["macros", "rt-multi-thread", "time"] }
```

Use `hydracache-core` directly only when building integrations or extensions
that need the shared core types.

## Current Functionality

HydraCache 0.1.0 includes:

- Local async cache runtime through `HydraCache`.
- Builder-based configuration through `HydraCache::local()`.
- Typed `put` and `get` operations.
- `get_or_load` for cache-aside loading.
- Per-entry TTL through `CacheOptions::ttl`.
- Tag-based invalidation through `CacheOptions::tags` and `invalidate_tag`.
- Key invalidation through `invalidate_key`.
- Full cache clearing through `flush`.
- Lightweight stats through `stats`.
- Compact default binary serialization through `PostcardCodec`.
- Custom serialization support through the `CacheCodec` trait.

## Basic Usage

```rust
use hydracache::{HydraCache, Options};
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
struct User {
    id: u64,
    name: String,
}

#[tokio::main]
async fn main() -> hydracache::CacheResult<()> {
    let cache = HydraCache::local().build();

    let user = User {
        id: 42,
        name: "Ada".to_owned(),
    };

    cache.put("user:42", &user, Options::new()).await?;

    let cached: Option<User> = cache.get("user:42").await?;
    assert_eq!(cached, Some(user));

    Ok(())
}
```

Values are serialized when inserted and deserialized when read. Types stored in
the cache must implement `serde::Serialize`, and types read from the cache must
implement `serde::de::DeserializeOwned`.

## Cache-Aside Loading

`get_or_load` checks the cache first. On a miss, it runs the loader, stores the
loaded value, and returns it.

```rust
use hydracache::{HydraCache, Options};
use serde::{Deserialize, Serialize};
use std::error::Error;
use std::fmt;

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
struct User {
    id: u64,
    name: String,
}

#[derive(Debug)]
struct LoadUserError;

impl fmt::Display for LoadUserError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        f.write_str("failed to load user")
    }
}

impl Error for LoadUserError {}

async fn load_user(id: u64) -> Result<User, LoadUserError> {
    Ok(User {
        id,
        name: format!("user-{id}"),
    })
}

#[tokio::main]
async fn main() -> hydracache::CacheResult<()> {
    let cache = HydraCache::local().build();

    let user = cache
        .get_or_load("user:42", Options::new(), || async { load_user(42).await })
        .await?;

    assert_eq!(user.id, 42);
    assert_eq!(cache.stats().loads, 1);

    Ok(())
}
```

In 0.1.0, concurrent misses are not deduplicated. If several tasks miss the
same key at the same time, each task may execute its own loader.

## TTL

Each entry can use a per-entry TTL. If no TTL is provided, the cache uses the
builder's default TTL.

```rust
use hydracache::{HydraCache, Options};
use serde::{Deserialize, Serialize};
use std::time::Duration;

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
struct Session {
    id: String,
}

#[tokio::main]
async fn main() -> hydracache::CacheResult<()> {
    let cache = HydraCache::local()
        .default_ttl(Duration::from_secs(300))
        .build();

    cache
        .put(
            "session:abc",
            &Session {
                id: "abc".to_owned(),
            },
            Options::new().ttl(Duration::from_secs(30)),
        )
        .await?;

    let cached: Option<Session> = cache.get("session:abc").await?;
    assert!(cached.is_some());

    Ok(())
}
```

Expired entries are removed lazily when read.

## Tag-Based Invalidation

Tags make it possible to invalidate groups of entries without knowing every key
at the call site.

```rust
use hydracache::{HydraCache, Options};
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
struct User {
    id: u64,
    name: String,
}

#[tokio::main]
async fn main() -> hydracache::CacheResult<()> {
    let cache = HydraCache::local().build();
    let options = Options::new().tags(["users", "tenant:acme"]);

    cache
        .put(
            "user:1",
            &User {
                id: 1,
                name: "Ada".to_owned(),
            },
            options.clone(),
        )
        .await?;

    cache
        .put(
            "user:2",
            &User {
                id: 2,
                name: "Grace".to_owned(),
            },
            options,
        )
        .await?;

    let removed = cache.invalidate_tag("users").await?;
    assert_eq!(removed, 2);

    let user: Option<User> = cache.get("user:1").await?;
    assert_eq!(user, None);

    Ok(())
}
```

`invalidate_tag` returns the number of entries removed for that tag.

## Key Invalidation And Flush

Use `invalidate_key` for a single entry and `flush` to clear the whole cache.

```rust
use hydracache::{HydraCache, Options};

#[tokio::main]
async fn main() -> hydracache::CacheResult<()> {
    let cache = HydraCache::local().build();

    cache.put("answer", &42_u64, Options::new()).await?;

    let removed = cache.invalidate_key("answer").await?;
    assert!(removed);

    cache.put("a", &1_u64, Options::new()).await?;
    cache.put("b", &2_u64, Options::new()).await?;
    cache.flush().await?;

    let value: Option<u64> = cache.get("a").await?;
    assert_eq!(value, None);

    Ok(())
}
```

## Stats

`stats` returns a snapshot of lightweight counters:

```rust
use hydracache::{HydraCache, Options};

#[tokio::main]
async fn main() -> hydracache::CacheResult<()> {
    let cache = HydraCache::local().build();

    let missing: Option<u64> = cache.get("missing").await?;
    assert_eq!(missing, None);

    cache
        .get_or_load("answer", Options::new(), || async {
            Ok::<_, std::convert::Infallible>(42_u64)
        })
        .await?;

    let stats = cache.stats();
    assert_eq!(stats.misses, 2);
    assert_eq!(stats.loads, 1);

    Ok(())
}
```

Counters in 0.1.0:

- `hits`: successful cache lookups.
- `misses`: lookups that did not return a usable value.
- `loads`: loader closures executed by `get_or_load`.
- `invalidations`: entries removed by invalidation APIs.
- `evictions`: reserved for backend eviction tracking; this remains zero in
  0.1.0.

## Builder Options

The local runtime can be configured before construction:

```rust
use hydracache::HydraCache;
use std::time::Duration;

let cache = HydraCache::local()
    .max_capacity(50_000)
    .max_entry_bytes(1024 * 1024)
    .default_ttl(Duration::from_secs(60))
    .build();
```

Builder settings:

- `max_capacity`: maximum weighted capacity for the Moka backend.
- `max_entry_bytes`: maximum accepted encoded entry size used for weighting.
- `default_ttl`: TTL used when `CacheOptions` does not provide one.
- `codec`: replaces the default codec.

## Core Types

`hydracache-core` exposes:

- `CacheKey`: application-provided logical cache keys.
- `CacheOptions`: per-entry TTL and tag options.
- `CacheStats`: lightweight counters.
- `CacheCodec`: trait for typed value serialization.
- `PostcardCodec`: default compact binary codec.
- `CacheError`: encode, decode, loader, and backend errors.
- `Result<T>`: HydraCache result alias.

Example:

```rust
use hydracache_core::{CacheKey, CacheOptions};
use std::time::Duration;

let key = CacheKey::from("user:42");
assert_eq!(key.as_str(), "user:42");

let options = CacheOptions::new()
    .ttl(Duration::from_secs(30))
    .tags(["users", "tenant:acme"]);

assert_eq!(options.tags_value(), &["users", "tenant:acme"]);
```

## Known Limitations

HydraCache 0.1.0 deliberately leaves several features for future releases:

- No distributed invalidation protocol.
- No SQLx or database query adapter.
- No macro crate.
- No single-flight loader deduplication.
- No backend eviction listener wiring.
- No persistent storage backend.

These limitations are part of the v0 scope. The current release is intended to
be a small, predictable local cache foundation.
