# HydraCache 0.5.0 Release Notes

> Status: in development.

HydraCache 0.5.0 adds a typed, namespaced local-cache view.

This release is still local-only. It does not include SQL adapters, proc macros,
distributed invalidation, cluster roles, persistence, or CDC integration.

## Published Crates

This release includes:

- `hydracache-core`
- `hydracache`

`hydracache-core` should be published first because `hydracache` depends on it.

## What Changed

### Typed Local Cache View

`HydraCache::typed::<T>("namespace")` creates a `TypedCache<T>` over the same
underlying cache.

```rust
let users = cache.typed::<User>("users");

let user = users
    .get_or_insert_with("42", CacheOptions::new(), || async {
        User {
            id: 42,
            name: "Ada".to_owned(),
        }
    })
    .await?;
```

The typed view prefixes keys as `namespace:key`, so `users:42` and `admins:42`
do not collide.

### Shared Runtime Guarantees

`TypedCache<T>` is a thin view, not a separate storage backend. It reuses:

- codec-backed `Bytes` storage
- TTLs
- tags
- local single-flight
- tag-generation invalidation safety
- stats

### Typed View Test Coverage

The test suite now covers:

- namespace isolation
- raw key and typed key separation
- nested namespaces
- typed loader helpers
- typed single-flight
- TTL expiration through typed views
- shared flush and stats
- loader errors
- tag invalidation through typed views
- stale load discard behavior through typed views

### Cargo Doc Examples

Public rustdoc examples were added for the main crate entry points and core
types. These examples are valid doctests and are included in the standard
documentation generated by `cargo doc`.

## Compatibility Notes

No existing API was removed.

The workspace version was bumped to `0.5.0`.
