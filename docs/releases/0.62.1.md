# HydraCache 0.62.1

`0.62.1` is a proof-cleanup patch over `0.62.0`. It does not add product
surface area; it closes the remaining evidence gaps in the cluster correctness
ledger.

## Highlights

- Added deterministic raft-filter tests for stale messages from a retired voter
  and delayed drain traffic around leader promotion.
- Added snapshot crash failpoint coverage for the persist-before-apply boundary.
- Expanded falsifiability canaries so `canary_raft_skip_save_conf_state` proves
  the drain/conf-state guard can turn red.
- Hardened the server promotion loop so recently removed raft voters are not
  automatically re-added before the metadata leave catches up.
- Reconciled the `0.62.0` release ledger wording and replaced the draft-only
  `canary_raft_disable_confchange_dedup` reference with the shipped canary map.

## Verification

```powershell
cargo test -p hydracache-cluster-raft --test raft_message_filter --locked
cargo test -p hydracache-cluster-raft --features test-failpoints --test failpoints_crash_safety --locked -- --test-threads=1
cargo test -p hydracache-server sync_raft_voters --locked
cargo check -p hydracache-server --all-targets --locked
```

The broader release gate remains the `0.62` fast proof block plus
`cargo xtask verify-no-test-features`.
