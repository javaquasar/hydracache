# HydraCache 0.19.0

Status: in development.

## Distributed Invalidation Bus

HydraCache now has the first synchronization layer for multiple cache instances:

- `CacheInvalidation`
- `CacheInvalidationMessage`
- `CacheInvalidationBus`
- `CacheInvalidationReceiver`
- `InMemoryInvalidationBus`
- `HydraCacheBuilder::shared_invalidation_bus(...)`
- `HydraCacheBuilder::invalidation_bus(...)`
- `HydraCacheBuilder::invalidation_node_id(...)`
- `HydraCache::invalidation_node_id()`

The bus propagates invalidation intent for `invalidate_key`, `remove`,
`invalidate_tag`, and `flush`. It does not replicate cached values.

Remote invalidations are applied locally with `CacheEventOrigin::DistributedBus`
and are not republished, which prevents echo loops.

## Diagnostics

`CacheStats` now includes bus counters:

- `distributed_invalidations_published`
- `distributed_invalidations_received`
- `distributed_invalidations_applied`
- `has_distributed_invalidation_activity()`

The observability snapshot and sandbox diagnostics include the same counters.

## Sandbox

The sandbox now exposes:

```text
POST /demo/distributed/invalidation/run
```

The endpoint creates two temporary cache nodes on one in-memory bus and verifies
remote tag invalidation, key invalidation, and flush propagation. The response
includes target remote events, source diagnostics, target diagnostics, and a
`passed` flag.

Swagger UI and the local dashboard include the new flow.

## Documentation

README now includes a distributed invalidation bus example with two cache
instances, a shared `InMemoryInvalidationBus`, remote event observation, and
diagnostic assertions.

The implementation plan is documented in:

```text
docs/plans/V0_19_DISTRIBUTED_INVALIDATION_BUS_PLAN.md
```

## Tests

New tests cover runtime propagation for tag, key, flush, typed physical keys,
and echo-loop suppression. Sandbox tests cover OpenAPI registration and the new
distributed invalidation endpoint.
