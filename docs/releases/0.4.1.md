# HydraCache 0.4.1 Release Notes

> Status: in development.

HydraCache 0.4.1 is a verification-focused patch release.

It does not change runtime behavior or public API. The release adds a small
`loom` model test suite for the critical tag-generation and in-flight freshness
logic introduced in `0.4.0`.

## Published Crates

This release includes:

- `hydracache-core`
- `hydracache`

`hydracache-core` should be published first because `hydracache` depends on it.

## What Changed

### Loom Invalidation Model

Added `loom` model tests for the core concurrency idea behind invalidation
safety:

- invalidation plus loader completion never leaves a stale value cached
- stale loader completion cannot overwrite a fresh value
- post-invalidation callers replace stale in-flight entries instead of joining them
- `flush` makes active load generations stale through the global generation

The model intentionally does not include `tokio`, `moka`, or the full
HydraCache runtime. It isolates the small concurrent state machine that matters
for freshness correctness.

### Dependency Refresh

Workspace dependency floors were refreshed to current compatible releases,
including `thiserror` 2.x.

## Compatibility Notes

No public API changes.

`loom` is a dev-dependency only.
