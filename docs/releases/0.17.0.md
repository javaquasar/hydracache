# HydraCache 0.17.0 Release Notes

HydraCache `0.17.0` focuses on the workspace-only sandbox. The sandbox is now a
small behavior lab for executable cache recipes, assertions, session replay
context, benchmark comparison, and local observability demos.

## Added

- JSON scenario document runner at `POST /demo/scenarios/document/run`.
- Small YAML scenario document parser at `POST /demo/scenarios/document/parse`.
- Committed scenario recipes:
  - `crates/hydracache-sandbox/scenarios/golden-path.json`
  - `crates/hydracache-sandbox/scenarios/golden-path.yaml`
- Scenario document step actions for user loads, user updates, user
  invalidation, raw cache put/get, TTL, single-flight, invalidation race,
  expected loader errors, and manual benchmark steps.
- Scenario assertions for passed/failed steps, loader calls, function calls,
  cache hits, misses, loads, single-flight joins, stale-load discards,
  invalidations, retained event count, and flow event count.
- Visual timeline section in `/demo/ui`.
- Benchmark diff endpoint at `POST /demo/benchmarks/compare`.
- Prometheus text demo endpoint at `GET /demo/observability/prometheus`.
- OpenTelemetry-style trace demo endpoint at
  `GET /demo/observability/traces/latest`.
- SQLite/Postgres schema and seed files under:
  - `crates/hydracache-sandbox/migrations/sqlite/`
  - `crates/hydracache-sandbox/migrations/postgres/`
  - `crates/hydracache-sandbox/seeds/`
- Seed report endpoint at `GET /demo/db/seed-report`.
- Session import endpoint at `POST /demo/import` for previously exported event
  streams.
- OpenAPI generated-client contract check at `GET /demo/openapi/client-check`.
- Minimal no-dependency fetch client example at
  `crates/hydracache-sandbox/openapi/generated-client.js`.
- README, HTTP collection, and PowerShell demo-flow updates for the new sandbox
  lab endpoints.

## Notes

- The YAML parser is deliberately small and supports the committed scenario DSL
  shape: top-level scalar fields plus `steps:` and `assertions:` lists.
- Session import restores event-log context only. It does not rebuild cache
  entries or backing-store rows from old events.
- Prometheus and trace endpoints are dependency-free teaching views over current
  sandbox diagnostics/events.
- `hydracache-sandbox` remains non-published.

## Validation

- `cargo check -p hydracache-sandbox --all-targets`
- `cargo test -p hydracache-sandbox --lib`

Run full workspace checks before tagging the release.
