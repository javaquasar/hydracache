# HydraCache 0.2.0 Release Notes

HydraCache 0.2.0 adds local single-flight miss deduplication to `get_or_load`.

This release is still local-only. It does not include SQL adapters, distributed
invalidation, cluster roles, or generation counters.

## Published Crates

This release includes:

- `hydracache-core`
- `hydracache`

`hydracache-core` should be published first because `hydracache` depends on it.

## What Changed

### Local Single-Flight

Concurrent `get_or_load` calls for the same missing key now share one loader
execution.

Before:

```text
caller A -> miss -> loader
caller B -> miss -> loader
caller C -> miss -> loader
```

Now:

```text
caller A -> miss -> loader
caller B -> wait for caller A
caller C -> wait for caller A
```

Cache hits bypass single-flight entirely.

### Error Sharing And Retry

When a shared loader returns an error, all current waiters receive the same
loader error.

The in-flight entry is removed after success or failure, so a later call can
retry normally.

### Tests

The test suite now covers:

- concurrent misses sharing one loader execution
- cached hits bypassing single-flight
- shared loader errors
- retry after loader error
- different keys running different loaders

## Compatibility Notes

`CacheError` is now cloneable internally so single-flight waiters can share
loader failures. Loader errors are stored as strings inside `CacheError::Loader`.

This should not affect normal application error handling through `?`, but code
that pattern-matched the exact internal loader error representation may need to
adjust.

## Still Out Of Scope

The following remain planned for later releases:

- SQLx adapter
- proc macros
- distributed invalidation
- cluster roles
- generation counters
- persistence
