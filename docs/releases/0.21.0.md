# HydraCache 0.21.0

Status: ready for release.

## Cluster Control Plane Contract

HydraCache now exposes `ClusterControlPlane`, a transport-neutral seam for
future cluster backends.

The trait covers:

- logical cluster name;
- invalidation bus selection after admission;
- member admission;
- client admission;
- node leave;
- role/node/generation/epoch diagnostics.

`InMemoryCluster` implements this trait, so existing `.shared_cluster(...)`
usage continues to work. New adapters can be passed through:

```rust
use std::sync::Arc;

use hydracache::{ClusterControlPlane, HydraCache};

# async fn example(control_plane: Arc<dyn ClusterControlPlane>) -> hydracache::CacheResult<()> {
let cache = HydraCache::client()
    .control_plane(control_plane)
    .node_id("api-client-a")
    .connect()
    .await?;

let diagnostics = cache.cluster_diagnostics().expect("cluster runtime");
assert_eq!(diagnostics.node_id.as_str(), "api-client-a");
# Ok(())
# }
```

This is intentionally still not a network implementation. It gives future
chitchat and raft-rs adapters a stable boundary to implement without changing
ordinary cache usage.

## Builder Behavior

Both cluster builders now support:

- `.shared_cluster(Arc<InMemoryCluster>)` for the embedded in-memory model;
- `.control_plane(Arc<dyn ClusterControlPlane>)` for custom adapters.

The builders announce candidates to `InMemoryClusterDiscovery` when configured,
then ask the control plane to admit the candidate. Admission errors are returned
to the caller.

## Tests

New tests cover:

- `InMemoryCluster` as a `ClusterControlPlane`;
- successful client/member builders using a trait-object control plane;
- custom control-plane admission failures for clients and members;
- continued compatibility for `.shared_cluster(...)`.
