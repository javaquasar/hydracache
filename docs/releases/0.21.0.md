# HydraCache 0.21.0

Status: in development.

0.21.0 focuses on cluster operability: stronger post-publish verification,
richer diagnostics, the first ownership resolver API, a transport-neutral peer
fetch seam, and a sandbox lab that makes the cluster flow easier to explain.

## Planned Scope

- Post-publish verification hardening for every published crate.
- Cluster diagnostics v2.
- Deterministic ownership resolver.
- Peer fetch design/API spike.
- Sandbox cluster lab scenario.
- Release hardening and documentation updates.

## Post-Publish Verification Hardening

The post-publish workflow now verifies the published crates as an external
consumer would use them. The smoke crate pulls `hydracache`,
`hydracache-core`, `hydracache-cluster`, `hydracache-cluster-chitchat`,
`hydracache-cluster-raft`, `hydracache-observability`,
`hydracache-actuator-axum`, `hydracache-db`, and `hydracache-sqlx` from
crates.io and exercises:

- local typed cache APIs;
- cluster composition through `HydraCluster`;
- real chitchat discovery and raft metadata crates;
- DB cache adapter aliases;
- framework-neutral observability snapshots;
- Axum actuator route construction.

## Non-Goals

- Production multi-node Raft transport.
- Durable metadata storage.
- Remote owner-side query execution.
- External invalidation transports.
- Production-grade security/authentication.

See [the 0.21.0 plan](../plans/V0_21_CLUSTER_OPERABILITY_PLAN.md) for the
implementation roadmap.
