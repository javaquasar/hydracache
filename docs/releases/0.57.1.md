# HydraCache 0.57.1

`0.57.1` is a focused maintenance release after `0.57.0`. It closes the
technical-debt items that could be closed before the endurance work in `0.58.0`
and explicitly leaves feature-sized debt for later releases.

The release does not add product surface area, a new consistency level, or a
new algorithm. Its purpose is dependency hygiene, supply-chain honesty, a
stronger operator lifecycle proof, and a reconciled debt ledger.

## Highlights

- Refreshed compatible dependency lockfile entries under the normal release
  gates.
- Scheduled and evaluated major dependency bumps one at a time, with consumer
  suites attached to each bump.
- Re-affirmed the raft/protobuf RustSec advisory state and kept the `deny.toml`
  ignore tied to a dated technical-debt rationale.
- Added the driven operator lifecycle kind E2E path used by later soak and
  operator-scale releases.
- Reconciled the technical-debt ledger: already-closed items were marked
  closed, and feature-sized items such as placement/autoscaling, Java artifacts,
  and networked daemon hosting stayed explicitly out of scope.
- Kept `doc-check`, release manifest status, and plan headers aligned so the
  shipped ledger remains executable.

## Verification

Representative gates:

```powershell
cargo deny check
cargo xtask doc-check
cargo xtask verify
```

Dependency-specific consumer suites were run for the crates touched by each
upgrade. The operator lifecycle proof remains kind-gated and skips loud when a
cluster is not available.

## Scope Boundary

`0.57.1` is housekeeping before endurance and networked-grid work. It does not
close TD-0004 placement/autoscaling, the future Java toolkit artifact branch,
TD-0008 networked daemon grid hosting, or the pre-release dependency migrations
that were intentionally left in the bucket-C backlog.
