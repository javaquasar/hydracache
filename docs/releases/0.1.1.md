# HydraCache 0.1.1 Release Notes

> Status: published.

HydraCache 0.1.1 is a patch release focused on API polish, documentation,
and stronger local-cache test coverage. It does not add SQL adapters,
distributed invalidation, or single-flight loader deduplication.

## Published Crates

This release includes:

- `hydracache-core`
- `hydracache`

`hydracache-core` should be published first because `hydracache` depends on it.

## What Changed

### API Polish

Added `CacheOptions::tag` for the common one-tag case:

```rust
let options = CacheOptions::new().tag("users");
```

Added `HydraCache::contains_key`:

```rust
if cache.contains_key("user:42").await {
    // The entry exists and has not expired.
}
```

Added `HydraCache::remove` as a shorter local-cache alias for key invalidation:

```rust
cache.remove("user:42").await?;
```

`invalidate_key` remains available and is still useful when code is written in
the same vocabulary as `invalidate_tag`.

### Tests

The test suite now covers:

- one-tag options through `CacheOptions::tag`
- `contains_key`
- `remove`
- overwriting an entry and cleaning old tag mappings
- per-entry TTL overriding the default TTL
- cloned cache handles sharing state

### Documentation

The README now documents the 0.1.1 polish APIs and clarifies the relationship
between `remove` and `invalidate_key`.

## Compatibility

This is intended to be source-compatible with 0.1.0 for existing v0 users.

No behavior was removed.

## Still Out Of Scope

The following remain planned for later releases:

- SQLx adapter
- proc macros
- distributed invalidation
- cluster roles
- single-flight
- generation counters
- persistence
