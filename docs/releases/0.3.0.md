# HydraCache 0.3.0 Release Notes

> Status: in development.

HydraCache 0.3.0 focuses on local-cache ergonomics and single-flight observability.

This release is still local-only. It does not include SQL adapters, distributed
invalidation, cluster roles, generation counters, or persistence.

## Published Crates

This release includes:

- `hydracache-core`
- `hydracache`

`hydracache-core` should be published first because `hydracache` depends on it.

## What Changed

### Local Loader Ergonomics

`get_or_insert_with` was added for infallible async loaders:

```rust
let user = cache
    .get_or_insert_with("user:42", CacheOptions::new(), || async {
        User {
            id: 42,
            name: "Ada".to_owned(),
        }
    })
    .await?;
```

`try_get_or_insert_with` was added as a fallible-loader spelling that mirrors
common cache-map APIs. It behaves the same as `get_or_load`.

### Single-Flight Observability

`CacheStats` now includes `single_flight_joins`.

This counter tracks calls that joined an already running single-flight load
instead of executing their own loader.

### Stronger Single-Flight Registration

The in-flight load map now performs check-or-insert under a write lock. This
avoids a race where two concurrent miss callers could both create a loader
future before either one registered it.

## Compatibility Notes

`CacheStats` has a new public field: `single_flight_joins`.

Most users reading stats by field name do not need to change anything. Code
that constructs `CacheStats` manually must provide the new field.

## Still Out Of Scope

The following remain planned for later releases:

- SQLx adapter
- proc macros
- distributed invalidation
- cluster roles
- generation counters
- persistence
