# HydraCache 0.31.1

> Status: patch release candidate.

`0.31.1` is a packaging and documentation polish release for the ORM adapter
work introduced in `0.31.0`. It does not change public APIs.

## Highlights

- Use crate-local `README.md` files for `hydracache-diesel` and
  `hydracache-seaorm` so `cargo publish` no longer warns about a README path
  outside the packaged crate.
- Keep the Diesel and SeaORM adapter READMEs focused on their specific query
  cache helpers instead of inheriting the root workspace README.
- Keep the expanded SQLx, Diesel, SeaORM, and sandbox edge-case coverage in
  the `0.31.x` verification baseline.

## Verification

```powershell
cargo fmt --all -- --check
cargo check --workspace --all-targets --locked
cargo test --workspace --all-targets --locked
cargo clippy --workspace --all-targets --all-features --locked -- -D warnings
cargo test --doc --workspace --locked
```

Package listing checks for the adapter crates should not emit the old
out-of-package README warning:

```powershell
cargo package -p hydracache-diesel --list --allow-dirty
cargo package -p hydracache-seaorm --list --allow-dirty
```
