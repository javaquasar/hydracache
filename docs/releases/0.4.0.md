# HydraCache 0.4.0 Release Notes

> Status: in development.

HydraCache 0.4.0 adds local tag-generation invalidation safety.

This release is still local-only. It does not include SQL adapters, distributed
invalidation, cluster roles, persistence, or CDC integration.

## Published Crates

This release includes:

- `hydracache-core`
- `hydracache`

`hydracache-core` should be published first because `hydracache` depends on it.

## What Changed

### Tag Generation Guards

Tagged loader calls now snapshot the current generation of their tags before
running.

If a tag is invalidated before the loader finishes, HydraCache returns the
loaded value to the original caller but does not store it back into the cache.

### Fresh In-Flight Join Semantics

Callers that arrive after `invalidate_tag` no longer join stale in-flight loads
for the same key. They start or join a fresh load under the new tag generation.

### Stale Load Stats

`CacheStats` now includes `stale_load_discards`.

This counter tracks loader results that were skipped because an invalidation
generation changed while the loader was running.

### Concurrency Stress Coverage

The test suite now includes stress scenarios for:

- many callers sharing one single-flight load
- concurrent tagged loads and invalidations
- mixed `put`, `remove`, `flush`, and `get_or_load` operations

## Compatibility Notes

`CacheStats` has a new public field: `stale_load_discards`.

Code that constructs `CacheStats` manually must provide the new field.

## Still Out Of Scope

The following remain planned for later releases:

- SQLx adapter
- proc macros
- distributed invalidation
- cluster roles
- persistence
- CDC-driven invalidation
