# HydraCache 0.6.0 Release Notes

> Status: in development.

HydraCache 0.6.0 adds key and tag ergonomics for safer local-cache usage.

This release is still local-only. It does not include SQL adapters, proc macros,
distributed invalidation, cluster roles, persistence, or CDC integration.

## Published Crates

This release includes:

- `hydracache-core`
- `hydracache`

`hydracache-core` should be published first because `hydracache` depends on it.

## What Changed

### CacheKeyBuilder

`CacheKeyBuilder` builds escaped `:`-separated keys from logical segments.

```rust
let key = CacheKeyBuilder::new()
    .tenant(7)
    .entity("user", 42)
    .build_string();
```

Segments escape `:` and `%`, so values containing separators do not accidentally
become multiple key segments.

### TagSet

`TagSet` collects reusable invalidation tags.

```rust
let tags = TagSet::new()
    .tag("users")
    .tenant(7)
    .entity("user", 42);

let options = CacheOptions::new().tag_set(tags);
```

### TypedCache Integration

`TypedCache::key_from` builds a physical namespaced key from a `CacheKeyBuilder`.

### Internal Module Split

The `hydracache` crate was split into focused modules for:

- cache runtime
- builder
- typed view
- entries
- in-flight loads
- tag index
- stats

The `hydracache-core` crate was split into focused modules for:

- codec
- errors
- keys
- options
- stats
- tags

This is a structural refactor only. Public re-exports and runtime behavior are
unchanged.

## Test Coverage

The new key/tag code is covered by:

- 12 core unit tests
- runtime integration tests for typed key builders and `TagSet` invalidation
- rustdoc examples

The refactor keeps the full runtime, loom, unit, doctest, clippy, and cargo-doc
checks green.

## Compatibility Notes

No existing API was removed.

The workspace version was bumped to `0.6.0`.
