# HydraCache 0.22.0

> Status: in progress.

0.22.0 adds the first real remote peer-fetch transport for cluster members.
The base runtime remains local-first; HTTP transport support lives in the
optional `hydracache-cluster-transport-axum` crate.

## Highlights

- Added `HydraCache::get_encoded(...)` for transport adapters that need to read
  stored codec bytes without knowing the application value type.
- Added publishable `hydracache-cluster-transport-axum`.
- Added `AxumPeerFetchService` with `POST /cluster/peer-fetch`.
- Added `HttpPeerFetch`, an HTTP implementation of `ClusterPeerFetch`.
- Added owner-id and generation checks for peer-fetch requests.
- Added route/client tests against a real local Axum server.
- Kept the MVP transport to plain HTTP/JSON; TLS/authentication remain explicit
  future work.

## Why This Matters

0.21.0 introduced deterministic ownership and a transport-neutral peer-fetch
seam. 0.22.0 makes that seam executable over HTTP while keeping the dependency
boundary clean:

- local-only users do not pull Axum or Reqwest;
- cluster users can expose encoded values from owner members;
- payloads remain codec-preserving bytes;
- stale owner generations are rejected instead of silently returning data from
  a restarted member.

## Example

```rust
use std::sync::Arc;

use hydracache::{CacheOptions, ClusterGeneration, HydraCache};
use hydracache_cluster_transport_axum::{AxumPeerFetchService, HttpPeerFetch};

# async fn example() -> hydracache::CacheResult<()> {
let owner_cache = HydraCache::local().build();
owner_cache.put("user:42", 42_u64, CacheOptions::new()).await?;

let routes = AxumPeerFetchService::new(
    "member-a",
    ClusterGeneration::new(1),
    Arc::new(owner_cache),
)
.routes();
# let _ = routes;

let peer_fetch = HttpPeerFetch::for_base_url("http://127.0.0.1:3000");
assert_eq!(
    peer_fetch.endpoint(),
    "http://127.0.0.1:3000/cluster/peer-fetch"
);
# Ok(())
# }
```

## Boundaries

This release does not yet add automatic remote owner-side loader/query
execution. The peer-fetch transport returns values already present in the owner
member's local cache. Future work can layer owner-side load-on-miss and
membership-aware routing on top of this transport.
