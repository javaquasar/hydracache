# HydraCache 0.53.1

`0.53.1` raises the interactive lab's election path from a modeled election to a
real `raft-rs` election on the native/server route. The simulator drives
`hydracache-cluster-raft` deterministically over the seeded `SimNetwork` and
`SimClock`, validates the model against real raft behavior, and exposes the
election source honestly in snapshots and UI.

The release does not change the production runtime. It improves the teaching
surface by using the shipped raft runtime where the host can support it.

## Highlights

- Added `SimRaftCluster` coverage that drives real `raft::RawNode`
  deterministically over the simulator network.
- Seeded raft randomized election timeouts so election runs are replayable.
- Added model-vs-raft validation so the existing modeled election stays checked
  against real raft behavior.
- Surfaced `election_source` and related disclosure fields in snapshots.
- Defaulted the server/sandbox lab route to real raft while keeping WASM honest
  about its validated-model fallback.
- Recorded the raft-in-WASM decision path in ADR documentation.
- Updated lab docs and UI disclosure so users can see whether election data is
  live raft or modeled.

## Verification

Focused gates:

```powershell
cargo test -p hydracache-sim --locked sim_raft
cargo test -p hydracache-sim --locked election_model_vs_raft
cargo test -p hydracache-sim --locked raft_snapshot
```

Full gate:

```powershell
cargo xtask verify
```

The release also used sandbox `/sim/*` route checks and demo-smoke validation
for the election-source disclosure path.

## Scope Boundary

This point release improves the lab, not the production consensus engine. WASM
compatibility remains explicit: native/server paths can use real raft, while
WASM may use a labelled validated model.

## Release Status

Published as `0.53.1` after the real-raft simulator and disclosure gates.
