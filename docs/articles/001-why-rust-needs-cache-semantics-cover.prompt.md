# Cover Prompt: 001 - Why Rust Needs Cache Semantics

Create a 16:9 editorial technology cover image for an article titled "Why Rust Needs Cache Semantics, Not Just Another Cache Map".

Visual direction:

- clean, polished infrastructure-engineering editorial style;
- no readable text, no logos, no mascots, no UI screenshots;
- central local cache runtime as a bright in-process core inside an abstract backend service;
- branching data paths showing typed values, TTL metadata, tag invalidation, and request coalescing as visual motifs, not labels;
- subtle database and network nodes in the background to imply Redis/database/service integration without making them the focus;
- local-first feeling: the central node should look fast, stable, and close to the application, while distributed signals remain secondary;
- color palette: graphite, deep teal, muted copper, warm amber highlights;
- crisp lighting, high contrast, professional Medium/LinkedIn article cover composition;
- leave enough calm negative space for platform cropping.

Avoid:

- readable words or pseudo-text;
- literal Rust logos;
- brand logos;
- cluttered dashboards;
- cartoon characters;
- one-note neon cyberpunk styling.
