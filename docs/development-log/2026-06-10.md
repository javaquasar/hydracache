# 2026-06-10

## HydraCache 0.19.0

- Added the first distributed synchronization primitive: a pluggable
  invalidation bus abstraction for key, tag, remove, and flush propagation.
- Added `InMemoryInvalidationBus` for tests, demos, and embedded multi-cache
  applications.
- Added remote invalidation diagnostics and `DistributedBus` event-origin flows.
- Added bus health diagnostics for lagged receivers, publish failures, and
  closed receiver streams.
- Extended the sandbox with `POST /demo/distributed/invalidation/run` so Swagger
  and the dashboard can demonstrate two temporary cache nodes synchronizing
  invalidations through one in-memory bus.
- Added a sandbox timeline for distributed invalidation flows so the response
  explains source publish, target apply, and diagnostics verification steps.
- Documented the release direction in
  `docs/plans/V0_19_DISTRIBUTED_INVALIDATION_BUS_PLAN.md` and
  `docs/releases/0.19.0.md`.

## Cluster Formation Research

- Analyzed the downloaded cluster/runtime sources in `../cluster_libs`.
- Captured the 0.20.0 cluster-formation direction in
  `docs/plans/V0_20_CLUSTER_FORMATION_LIBRARY_ANALYSIS.md`.
- Captured the primary cluster/client development roadmap in
  `docs/plans/V0_20_CLUSTER_CLIENT_ROADMAP.md`.
- Recommended a layered design: discovery finds candidates, Raft commits
  membership/control-plane state, and the invalidation bus remains off the Raft
  hot path.

## HydraCache 0.20.0

- Started the first public cluster role API with `HydraCache::client()` and
  `HydraCache::member()`.
- Added `InMemoryCluster` as a test/demo runtime for client/member admission,
  stale generation rejection, simulated epochs, membership events, and shared
  invalidation propagation.
- Added `InMemoryClusterDiscovery` as a dependency-free discovery journal for
  candidate and liveness events before authoritative admission.
- Added cluster diagnostics so a cache can report role, node id, generation,
  epoch, bootstrap metadata, member/client counts, and invalidation subscriber
  count.
- Added tests for the in-memory discovery journal, in-memory cluster model,
  stale generation protection, builder flows, and member/client invalidation
  propagation.
- Documented the release in `docs/releases/0.20.0.md` and linked the 0.20
  cluster research plans from the README.

## HydraCache 0.20.0 Control Plane Follow-Up

- Added `ClusterControlPlane` as the transport-neutral adapter seam for future
  chitchat/Raft-backed cluster implementations.
- Kept `InMemoryCluster` as the default embedded control plane while allowing
  `.control_plane(Arc<dyn ClusterControlPlane>)` on client/member builders.
- Preserved `.shared_cluster(Arc<InMemoryCluster>)` as the ergonomic in-memory
  path for tests, demos, and embedded apps.
- Added tests proving the in-memory control plane contract, trait-object
  builder usage, and custom adapter admission errors.
- Documented the feature in `docs/releases/0.20.0.md` and captured the plan in
  `docs/plans/V0_20_CLUSTER_CONTROL_PLANE_PLAN.md`.

## HydraCache 0.20.0 Discovery Adapter Follow-Up

- Added `ClusterDiscovery` as the transport-neutral adapter seam for future
  chitchat, DNS, mDNS, or P2P-backed discovery implementations.
- Kept `InMemoryClusterDiscovery` as the default embedded discovery journal
  while allowing `.discovery(Arc<dyn ClusterDiscovery>)` on client/member
  builders.
- Added discovery error semantics: failed candidate announcement stops builder
  startup before control-plane admission.
- Added `cluster_discovery_diagnostics()` so client/member runtimes can expose
  discovery candidates and event history when discovery is configured.
- Added tests proving the in-memory discovery contract, trait-object builder
  usage, custom discovery errors before admission, and discovery diagnostics.
- Documented the feature in `docs/releases/0.20.0.md` and captured the plan in
  `docs/plans/V0_20_CLUSTER_DISCOVERY_ADAPTER_PLAN.md`.

## HydraCache 0.20.0 Lifecycle Follow-Up

- Added `HydraCache::leave_cluster().await` for explicit client/member cluster
  leave.
- Local caches return `Ok(None)`, while admitted client/member runtimes return
  the control-plane `ClusterMembershipEvent`.
- Added tests proving client and member leave behavior, repeated leave behavior,
  and local cache contents surviving membership removal.
- Documented the lifecycle behavior in `docs/releases/0.20.0.md`.

## HydraCache 0.20.0 Sandbox Cluster Demo

- Added `POST /demo/cluster/lifecycle/run` to the workspace-only sandbox.
- The route uses the public client/member API to demonstrate discovery,
  admission, member-to-client invalidation, client-to-member invalidation,
  explicit leave, and retained local cache contents.
- Registered OpenAPI schemas and dashboard/report entries so the flow is visible
  from Swagger UI and `/demo/ui`.
- Extended sandbox route tests to call the new endpoint and assert the timeline,
  membership reports, discovery report, and remote events.

## HydraCache 0.20.0 Multi-Node Cluster Regression

- Added a two-member/two-client cluster regression test.
- Verified member-to-all tag invalidation and client-to-all key invalidation
  through the shared distributed invalidation bus.
- Verified generation upgrade plus stale-generation rejection in a larger
  membership set.
- Verified explicit leave after distributed activity updates member/client
  counts without breaking diagnostics.
