# 2026-04-29

## Summary

HydraCache's product direction was clarified:

- local cache first
- database result caching through adapters
- distributed synchronization later
- explicit cluster roles: `Local`, `Client`, `Member`

## Important Changes

- The architecture document now treats `sqlx` as the first DB adapter, not the whole product identity.
- The API direction separates local cache APIs from query adapter APIs.
- The key model moved from SQL-specific `sql_hash` / `sql_str` language to `namespace_hash` / `logical_id`.
- Phase 0 now emphasizes a useful local cache product before query macros.
- Generation counters are now recommended as an optional Phase 1 mitigation for invalidation/load races.
