# 2026-06-04

## 0.15.0 Implementation Notes

Started HydraCache `0.15.0` with cacheable macro ergonomics:

- Added `tags = ...` support to `cacheable!(...)`.
- Kept repeated `tag = ...` support for compatibility.
- Added `cacheable_infallible!(...)` over `HydraCache::get_or_insert_with`.
- Re-exported `cacheable_infallible!` from `hydracache`.
- Added runtime tests for array tags, `TagSet`, and infallible cache hits.
- Added trybuild compile-pass tests for the new public macro forms.
- Added `crates/hydracache/examples/cacheable_function.rs` as the live usage
  example for ordinary function caching.

## 0.16.0 Implementation Notes

Started HydraCache `0.16.0` with local observability ergonomics:

- Added `CacheStats` helpers for total requests, hit ratio, single-flight
  activity, and stale-load discard checks.
- Added `CacheDiagnostics` as a small smoke-test snapshot for stats plus the
  local backend's approximate entry count.
- Added `HydraCache::diagnostics().await` and `TypedCache::diagnostics().await`.
- Added runtime tests that verify repeated cached calls report one load, one
  hit, and a non-empty diagnostic snapshot.
- Expanded README and cargo-doc examples with practical "did the cache work?"
  checks.

## 0.16.0 Actuator Implementation Notes

Extended HydraCache `0.16.0` with opt-in actuator modules:

- Added `hydracache-observability` as a framework-neutral registry and
  serializable diagnostics crate.
- Added `hydracache-actuator-axum` as a read-only Axum route adapter.
- Exposed health, cache list, overview, per-cache diagnostics, and per-cache
  stats endpoints.
- Kept HTTP dependencies out of the base `hydracache` crate.
- Deferred write-enabled actuator endpoints until there is an explicit security
  model.
- Added non-published `hydracache-sandbox` manual backend with memory,
  SQLite-memory, SQLite-file, and Postgres-Docker modes.
- Added Swagger-compatible sandbox UI and OpenAPI JSON for manual endpoint
  exploration.
- Added a committed sandbox `.env` demo profile and `.env.example` reference so
  local manual runs can start with `cargo run -p hydracache-sandbox`, while CLI
  flags remain available as overrides.
- Added sandbox profiles, generated OpenAPI through `utoipa`, local embedded
  Swagger UI assets through `utoipa-swagger-ui`, HTTP collection/scripted demo
  flows, and an optional Postgres Docker smoke test with graceful skip.
- Expanded the sandbox Swagger API into an interactive cache lab covering raw
  local-cache operations, typed-cache namespacing, database-backed query
  caching, cached non-database functions, TTL expiry, single-flight, and
  invalidation/load race reports.
- Added `postgres-compose` sandbox profile plus Docker Compose files for
  running either only the local Postgres dependency or the full sandbox API
  stack.
- Added a no-CDN sandbox developer console at `/demo/ui`, readiness endpoint,
  structured event log, reset endpoint, expected-failure negative scenarios,
  unified Compose profiles, and `Dockerfile.sandbox` for prebuilt local API
  images.
- Hardened the sandbox into a reproducible manual lab with `/demo/config`,
  `/demo/presets`, `/demo/export`, `POST /demo/self-test`, event filters,
  `flow_id` correlation, optional JSONL event persistence, mini UI counters,
  and Docker Compose healthchecks for the sandbox API service.
