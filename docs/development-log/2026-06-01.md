# 2026-06-01

## Summary

Started HydraCache `0.9.0` with database query API ergonomics. The main goal is
to reduce repeated key/tag boilerplate while preserving the explicit cache
boundary as the source of truth.

## 0.9.0 Implementation Notes

- Bumped the workspace version to `0.9.0`.
- Added `DbCache::entity` and `DbCache::collection` as database query ergonomics
  over the explicit descriptor API.
- Added `DbQuery::for_entity` and `DbQuery::collection_tag` for composing
  generated entity keys/tags with manual query descriptors.
- Kept explicit `.key(...)`, `.tag(...)`, `.ttl(...)`, and `.fetch_with(...)`
  as the full-control API and future macro expansion target.
- Extended unit tests for escaped generated keys/tags, explicit key override,
  tag preservation, cache hits, and invalidation.
- Extended the Postgres testcontainers test to verify the new helpers against a
  real SQLx/Postgres flow.
- Added release notes in [0.9.0.md](../releases/0.9.0.md).
