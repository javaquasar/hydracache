# Development Log - 2026-06-12

## 0.31.0 Adapter Parity

- Added the `0.31.0` plan for Diesel and SeaORM database-result cache adapters.
- Added `hydracache-diesel` as a Diesel-facing integration crate over the
  database-neutral `DbCache` API.
- Added `hydracache-seaorm` as a SeaORM-facing integration crate over the same
  metadata and cache descriptor model.
- Kept `hydracache-db` database-neutral. SQLx, Diesel, and SeaORM remain
  optional adapter crates instead of dependencies of the core query-cache API.
- Added a sandbox/OpenAPI ORM comparison endpoint:
  `POST /demo/query/users/{id}/orm-comparison`.

## Design Notes

- Diesel helpers run blocking loaders with `tokio::task::spawn_blocking`.
- SeaORM helpers accept async loaders directly.
- The sandbox comparison endpoint uses one selected sandbox backing row and
  three adapter namespaces: `orm-sqlx`, `orm-diesel`, and `orm-seaorm`.
- The comparison asserts the common cache-layer invariant: first call uses the
  loader, second call hits cache, and all adapters return the same row.

## Validation

- `cargo test -p hydracache-diesel -p hydracache-seaorm`
- `cargo test -p hydracache-sandbox --lib`
