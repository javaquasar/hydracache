# 2026-05-27

## Summary

HydraCache `0.1.1` was published as a patch release.

## 0.1.1 Published Scope

- `CacheOptions::tag`
- `HydraCache::contains_key`
- `HydraCache::remove`
- stronger tests for local cache polish
- release notes in [0.1.1.md](../releases/0.1.1.md)

## Next Milestone

Start `0.2.0` with local single-flight miss deduplication.

Goal:

- concurrent `get_or_load` calls for the same missing key should share one loader execution
- cache hits should bypass single-flight entirely
- in-flight entries should be cleaned up after success or error
- distributed and SQLx features remain out of scope

## 0.2.0 Implementation Notes

- Added local single-flight for `get_or_load`.
- Concurrent misses for the same key share one loader execution.
- Cache hits bypass single-flight.
- Loader errors are shared by current waiters and the in-flight entry is cleaned up afterward.
- Added release notes in [0.2.0.md](../releases/0.2.0.md).
- Published `0.2.0`; CI is green.

## CI Notes

- Added a dedicated GitHub Actions job for the declared MSRV: Rust `1.85.0`.
- Rust `1.85.0` was the initial floor because the committed dependency graph included crates requiring Cargo's Rust 2024 edition support.
- The MSRV job runs `cargo check --workspace --all-targets --locked` and `cargo test --workspace --locked`.

## MSRV Update Notes

- Raised the workspace MSRV to Rust `1.88.0` during the `0.7.0` SQLx/testcontainers work.
- Removed the need for Rust `1.85` transitive dependency pins around ICU/IDNA and testcontainers.
- Updated the CI MSRV job to Rust `1.88.0`.

## Next Milestone

Start `0.3.0` with local-cache ergonomics and observability polish.

Goal:

- keep HydraCache useful as a standalone local async cache
- make common local-cache flows easier to read and harder to misuse
- expose enough single-flight visibility to debug load storms
- prepare the API shape for later DB query adapters without making SQLx the public center of the library

## 0.3.0 Implementation Notes

- Bumped the workspace version to `0.3.0`.
- Added `get_or_insert_with` for infallible local-cache loaders.
- Added `try_get_or_insert_with` as the fallible local-cache spelling for `get_or_load`.
- Added `single_flight_joins` to `CacheStats`.
- Tightened the single-flight in-flight map so check-or-insert happens under a write lock.

## 0.4.0 Implementation Notes

- Bumped the workspace version to `0.4.0`.
- Added tag-generation snapshots for tagged loads.
- Prevented stale loader results from being stored after `invalidate_tag`.
- Prevented post-invalidation callers from joining stale in-flight loads.
- Added `stale_load_discards` to `CacheStats`.
- Added targeted concurrency stress tests for single-flight, tag invalidation, flush, put, remove, and load combinations.
- Documented the decision in [ADR-0002](../adr/ADR-0002-tag-generation-invalidation-safety.md).
- Published `0.4.0`.

## 0.4.1 Implementation Notes

- Bumped the workspace version to `0.4.1`.
- Added `loom` as a dev-dependency.
- Added a compact `loom` model for tag-generation and in-flight freshness.
- Verified stale store, stale overwrite, stale in-flight replacement, and flush generation interleavings.
- No runtime behavior or public API changes.

## 0.5.0 Implementation Notes

- Bumped the workspace version to `0.5.0`.
- Added `TypedCache<T>` as a typed, namespaced view over `HydraCache`.
- Added `HydraCache::typed::<T>("namespace")`.
- Added typed `get`, `put`, loader helpers, key removal, `contains_key`, shared tag invalidation, shared `flush`, and shared stats.
- Kept the storage model unchanged: typed views reuse the same codec-backed `Bytes` runtime.
- Expanded typed-cache tests for namespaces, TTL, shared flush/stats, loader errors, raw-key isolation, nested namespaces, single-flight, tag invalidation, and freshness races.
- Added public rustdoc examples for `HydraCache`, `TypedCache`, builder usage, loader helpers, and core types; verified them with doctests and `cargo doc`.

## 0.6.0 Implementation Notes

- Bumped the workspace version to `0.6.0`.
- Added `CacheKeyBuilder` with escaped key segments.
- Added tenant/entity helpers for key construction.
- Added `TagSet` with tenant/entity helpers.
- Added `CacheOptions::tag_set`.
- Added `TypedCache::key_from` for namespaced physical keys from builders.
- Covered all new key/tag code paths with unit, runtime, and doctest coverage.
- Split `hydracache` runtime code out of `lib.rs` into focused modules for cache runtime, builder, typed view, entries, in-flight loads, tag index, and stats.
- Split runtime tests into focused modules for local cache, typed cache, invalidation safety, and concurrency.
- Split `hydracache-core` out of `lib.rs` into focused modules for codec, error, key, options, stats, and tags while preserving public re-exports.

## 0.7.0 Implementation Notes

- Bumped the workspace version to `0.7.0`.
- Added `hydracache-db` as the database-neutral query result-cache adapter crate.
- Kept `hydracache-sqlx` as the SQLx-facing integration crate over `hydracache-db`.
- Added `DbCache` and `DbQuery` descriptors with explicit cache keys, optional diagnostic names, tags, and per-query TTL.
- Kept `SqlxCache` and `SqlxQuery` as SQLx-specific aliases over `DbCache` and `DbQuery`.
- Added `fetch_with` so SQLx remains responsible for pools, transactions, macros, and row mapping while HydraCache owns cache lookup, single-flight, storage, and invalidation.
- Added SQLx adapter tests for missing keys, metadata, cache hits, tag invalidation, and empty namespaces.
- Updated post-publish verification to install and smoke-test `hydracache-db` and `hydracache-sqlx` from crates.io.
