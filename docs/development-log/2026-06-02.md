## 0.12.0 Implementation Notes

Started HydraCache `0.12.0` with reusable database query cache policies:

- Added `QueryCachePolicy` in `hydracache-db`.
- Moved `DbQuery` internals to the policy object.
- Added `DbCache::cached_with::<T>(policy)`.
- Added `DbQuery::with_policy(policy)` and `DbQuery::cache_policy()`.
- Added `DbQuery::collection(name)`.
- Added `DbQuery::load(...)` as a repository-style alias for
  `fetch_with(...)`.
- Re-exported `QueryCachePolicy` from `hydracache-sqlx`.
- Added tests for policy metadata, reuse, replacement, collection policy, SQLx
  re-export, and the new `load(...)` cache-hit path.

## 0.13.0 Implementation Notes

Started HydraCache `0.13.0` with the first ergonomic macro on top of
`QueryCachePolicy`:

- Added `query_cache_policy!(...)` in `hydracache-macros`.
- Re-exported the macro from `hydracache-db` and `hydracache-sqlx`.
- Supported entity/id, explicit key, and collection key sources.
- Supported name, repeated tags, repeated collection tags, `ttl`, and
  `ttl_secs`.
- Required exactly one key source to avoid generating policies that fail later
  with missing-key errors.
- Added parser/expansion unit tests, `trybuild` pass/fail cases, and SQLx
  re-export coverage.

## 0.14.0 Idea Notes

Captured the next direction as a cacheable function API for ordinary expensive
functions, separate from database result caching:

- `hydracache` should own local function caching.
- `hydracache-db` should remain focused on database/repository result cache
  semantics.
- `hydracache-sqlx` should remain only the SQLx adapter.
- First implementation should likely be an explicit function-like macro or
  helper before attempting an attribute macro.

## 0.14.0 Implementation Notes

Started HydraCache `0.14.0` with a local-cache `cacheable!(...)` macro:

- Added `cacheable!` in `hydracache-macros`.
- Re-exported `cacheable!` from the base `hydracache` crate.
- Kept the macro explicit: `cache`, `key`, and `load` are required.
- Supported repeated `tag`, plus one of `ttl` or `ttl_secs`.
- Generated `HydraCache::get_or_load(...)` calls instead of introducing a new
  runtime path.
- Added async integration tests for cache hits, TTL expiration, and tag
  invalidation.
- Added `trybuild` compile-pass and compile-fail tests for downstream macro
  behavior.
- Verified workspace checks, docs, coverage, and package verification for all
  five crates.
